# Архитектурный Реестр: Этап 0 (Завершено)

## 🏗 Система: Core Query Builder (@void/core-query-builder)
- **Логика:** Fluent API для генерации SQL (SELECT, INSERT, UPDATE, DELETE).
- **Безопасность:** Параметризация запросов ($1, $2) для защиты от SQL-инъекций.
- **Качество:** 100% TDD (8 тестов), нативный `node:test`.
- **Стэк:** Pure JS (ESM), JSDoc, Node.js 20.16.0.

## 🛠 Инфраструктура
- **Monorepo:** NPM Workspaces (связка пакетов и приложений).
- **Docker:** Единый Dockerfile для всей экосистемы, тесты проходят в контейнере.

## 🧠 Snapshot знаний (для интервью)
1. **Почему не ORM?** Контроль над SQL, понимание планов запросов (Explain), отсутствие лишних абстракций.
2. **Зачем плейсхолдеры?** Чтобы отделить структуру запроса от данных. Драйвер БД сам подставит значения, исключая возможность инъекции.
3. **Зачем Docker на этапе 0?** Гарантия того, что "работает на моей машине" = "работает везде".

# Архитектурный Реестр: Этап 1 (Auth Service — Завершено)

## 🏗 Система: Auth Service (@void/auth-service)
- **Статус:** Завершено (Логика, Инфраструктура и HTTP API).
- **Domain Layer:**
  - Сущность `User`: инкапсулирует валидацию email (RegExp) и хранение хэша.
  - Инвариант: запрет на создание пользователя с некорректным форматом почты.
- **Use Case Layer:**
  - Сценарий `RegisterUser`: оркестратор регистрации (хэширование -> Entity -> Repo).
  - Сценарий `LoginUser`: аутентификация (поиск -> верификация -> генерация токенов).
- **Infrastructure Layer:**
  - `UserRepository`: Data Mapper для `core-query-builder`. Маппинг `camelCase` JS -> `snake_case` SQL.
  - `PasswordService`: Argon2id (OWASP Standard).
  - `TokenService`: Генерация и верификация JWT.
  - `SessionRepository`: Управление сессиями в Redis (TTL, Logout logic).
  - `Router`: Нативный HTTP-диспетчер с Dependency Injection.

## 🛠 Интеграция и Сеть
- **Docker Networking:** Сервис слушает `0.0.0.0`. Связь с БД и Redis через DNS-имена сервисов.
- **Security:** Реализован гибридный подход (Stateless JWT + Stateful Sessions в Redis).

## 🧠 Snapshot знаний (для интервью)
1. **Почему Argon2id?** Защита от GPU/ASIC атак, memory-hard алгоритм. Выбор для Senior-решений.
2. **В чем роль Use Case?** "Дирижер", который не зависит от деталей реализации (БД/Криптография).
3. **Зачем `0.0.0.0` в Docker?** Чтобы сервер принимал трафик с внешнего интерфейса контейнера, а не только внутри loopback.
4. **Зачем Redis для JWT?** Возможность мгновенного отзыва токена (Revocation), которую чистый JWT не обеспечивает.
